# Contributing to Plantry

Thank you for your interest in contributing to Plantry!

## Project Goals

Plantry is a **portfolio project** created during my IT Systems Integration apprenticeship.
While contributions are welcome, please understand:

- This is a learning project
- All PRs require review and approval
- Major changes should be discussed first via Issues

## Bug Reports

Found a bug? Please open an Issue with:
- Device & OS version
- Steps to reproduce
- Expected vs actual behavior
- Screenshots if possible

## Feature Requests

Have an idea? Open an Issue first to discuss before coding.

Include:
- Use case / problem you're solving
- Proposed solution
- Why it's valuable for growers

## Pull Requests

### Before You Start
1. Fork the repository
2. Create a feature branch (`git checkout -b feature/amazing-feature`)
3. Make your changes
4. Test thoroughly
5. Commit with clear messages

### PR Guidelines
- One feature/fix per PR
- Follow Dart/Flutter style guide
- Run `flutter analyze` before submitting
- Run `dart format .` before submitting
- Add tests for new features (if applicable)

### Code Style
```bash
# Format code
dart format .

# Analyze
flutter analyze

# Run tests
flutter test
```

## What We Don't Accept

- Breaking changes without discussion
- Code without testing
- Spam / low-effort PRs
- Features that require cloud/online services
- Features that compromise privacy

## Priority Areas

- Bug fixes
- Performance improvements
- Translation improvements (DE/EN)
- Test coverage
